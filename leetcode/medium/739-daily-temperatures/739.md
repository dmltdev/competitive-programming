# Daily Temperatures - Monotonic Stack

## Problem
For each day, find how many days until a warmer temperature. Return 0 if none exists.

## Intuition
Brute force: For each day, scan forward until finding a warmer day → O(n²)

Better: Use a stack to remember days waiting for warmer temperatures. When we find a warmer day, resolve all colder days in the stack.

## Algorithm
**Monotonic Decreasing Stack** - stores indices in decreasing temperature order

```
1. For each temperature (left to right):
   - While current temp > stack top temp:
     * Pop index from stack
     * Answer[popped] = current_index - popped_index
   - Push current index to stack
2. Remaining stack indices have no answer (stay 0)
```

## Example: [73,74,75,71,69,72,76,73]

```
i=0, temp=73: stack=[0]
i=1, temp=74: 74>73, pop 0, ans[0]=1, stack=[1]
i=2, temp=75: 75>74, pop 1, ans[1]=1, stack=[2]
i=3, temp=71: stack=[2,3]
i=4, temp=69: stack=[2,3,4]
i=5, temp=72: 72>69 pop 4 ans[4]=1, 72>71 pop 3 ans[3]=2, stack=[2,5]
i=6, temp=76: 76>72 pop 5 ans[5]=1, 76>75 pop 2 ans[2]=4, stack=[6]
i=7, temp=73: stack=[6,7]

Result: [1,1,4,2,1,1,0,0]
```

## Code
```typescript
function dailyTemperatures(temperatures: number[]): number[] {
  const ans: number[] = new Array(temperatures.length).fill(0);
  const stack: number[] = [];

  for (let i = 0; i < temperatures.length; i++) {
    while (stack.length > 0 && temperatures[i] > temperatures[stack[stack.length - 1]]) {
      const prevIndex = stack.pop()!;
      ans[prevIndex] = i - prevIndex;
    }
    stack.push(i);
  }

  return ans;
}
```

## Key Points
- Stack stores **indices** (not temps) - need indices to calculate distance
- Each element pushed once, popped once → O(n) time
- Stack maintains unsolved positions in decreasing temp order

## Complexity
- Time: O(n)
- Space: O(n)

## Pattern
**Monotonic Stack** - recognize when:
- Finding next greater/smaller element
- Brute force needs nested loops
- Need to track "waiting" positions
